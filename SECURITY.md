# Security Policy

## Reporting a vulnerability

If you believe you have found a security vulnerability in agenthatch, report it
privately through GitHub Security Advisories:

https://github.com/agenthatch/agenthatch/security/advisories/new

Do not report security vulnerabilities through public GitHub issues,
discussions, or pull requests.

When submitting a report, include as much relevant information as you can safely
provide:

- A description of the vulnerability
- Steps to reproduce, if safe to share privately
- Affected components, versions, or configurations
- Potential impact
- Any suggested mitigation or fix, if available

The maintainer will review reports submitted through GitHub Security Advisories
and coordinate follow-up there. Response times may vary depending on severity.

## Public disclosure

Do not publicly disclose details of a vulnerability until it has been reviewed
and, where appropriate, a fix or mitigation has been made available.

## Contact

For urgent security matters that cannot be reported through GitHub, contact
**eternalrights@outlook.com**.
